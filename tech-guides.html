1. Building Responsive Web Applications

  • Use CSS Grid and Flexbox - Modern layout systems for complex designs
  • Implement mobile-first design - Start with mobile, scale up for desktop

2. RESTful API Design Best Practices

  • Use meaningful HTTP methods - GET (read), POST (create), PUT (update), DELETE (remove)
  • Implement proper status codes - 200 (OK), 201 (Created), 400 (Bad Request), 404 (Not Found), 500 (Server Error)
  • Use nouns for resources - /users, /products, /orders (not /getUsers)
  • Version your APIs - /api/v1/resource for backward compatibility
  • Implement pagination - Use limit and offset for large datasets
  • Use JWT for authentication - Stateless, secure token-based auth
  • Document your API - Use Swagger/OpenAPI for documentation
// RESTful API Endpoints Example
GET    /api/v1/users           // Get all users
GET    /api/v1/users/:id       // Get single user
POST   /api/v1/users           // Create new user

3. JavaScript Performance Optimization

  • Minimize DOM manipulations - Batch updates, use DocumentFragment
  • Use debouncing and throttling - Limit function execution rate
  • Implement code splitting - Load only needed code chunks
  • Use Web Workers - Offload CPU-intensive tasks
  • Optimize loops - Cache array length, avoid nested loops
  • Cache DOM queries - Store references instead of querying repeatedly
  • Use requestAnimationFrame - Smooth animations, better performance
// Debounce Function
function debounce(func, wait) {
  let timeout;
  return function executedFunction(...args) {
    const later = () => {
      clearTimeout(timeout);
      func(...args);
    };
    clearTimeout(timeout);
    timeout = setTimeout(later, wait);
  };
}

// Throttle Function
function throttle(func, limit) {
  let inThrottle;
  return function(...args) {
    if (!inThrottle) {
      func.apply(this, args);
      inThrottle = true;
      setTimeout(() => inThrottle = false, limit);
    }
  };
}

// Usage
const handleResize = debounce(() => {
  console.log('Resized!');
}, 250);
window.addEventListener('resize', handleResize);

# Docstring Example
def calculate_average(numbers: list[float]) -> float:
    """
    Calculate the average of a list of numbers.
    
    Args:
        numbers: List of numeric values
        
    Returns:
        The average value as a float
        
    Raises:
        ValueError: If the list is empty
    """
    if not numbers:
        raise ValueError("List cannot be empty")
    return sum(numbers) / len(numbers)

# List Comprehension

even_squares = [x**2 for x in range(10) if x % 2 == 0]

# Type Hints
from typing import List, Dict, Optional

def process_data(
    items: List[Dict[str, any]],
    threshold: Optional[float] = None
) -> List[Dict]:
    return [item for item in items if item.get('value', 0) > (threshold or 0)]

5. Git Version Control Workflow

  • Use meaningful commit messages - Describe what and why, not how
  • Create feature branches - One branch per feature/bugfix
  • Use pull requests - Code review before merging
  • Keep commits small and focused - One logical change per commit
  • Use .gitignore - Exclude unnecessary files
  • Tag releases - Mark version milestones
  • Write good README - Project documentation

6. Database Design Principles

  • Normalize data - Reduce redundancy, improve integrity
  • Use appropriate data types - Optimize storage and performance
  • Create indexes - Speed up frequently queried fields
  • Implement foreign keys - Enforce referential integrity
  • Use transactions - Ensure data consistency
  • Backup regularly - Prevent data loss
  • Use connection pooling - Optimize database connections
-- Database Schema Example (PostgreSQL)

CREATE TABLE users (
  id SERIAL PRIMARY KEY,
  email VARCHAR(255) UNIQUE NOT NULL,
  username VARCHAR(50) UNIQUE NOT NULL,
  password_hash VARCHAR(255) NOT NULL,
  created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
  updated_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);

-- Create index for frequently queried field
CREATE INDEX idx_users_email ON users(email);

CREATE TABLE posts (
  id SERIAL PRIMARY KEY,
  user_id INTEGER REFERENCES users(id) ON DELETE CASCADE,
  title VARCHAR(255) NOT NULL,
  content TEXT,
  published BOOLEAN DEFAULT false,
  created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);

-- Create composite index
CREATE INDEX idx_posts_user_published ON posts(user_id, published);

-- Transaction Example
BEGIN;
  UPDATE accounts SET balance = balance - 100 WHERE id = 1;
  UPDATE accounts SET balance = balance + 100 WHERE id = 2;
COMMIT;

7. Security Best Practices

  • Never store plain text passwords - Use bcrypt, argon2, or scrypt
  • Validate and sanitize inputs - Prevent SQL injection, XSS attacks
  • Use HTTPS everywhere - Encrypt all communications
  • Implement CORS correctly - Restrict allowed origins
  • Use environment variables - Never hardcode secrets
  • Keep dependencies updated - Fix known vulnerabilities
  • Implement rate limiting - Prevent brute force attacks
  • Use security headers - CSP, X-Frame-Options, HSTS
// Password Hashing with bcrypt (Node.js)
const bcrypt = require('bcrypt');
const saltRounds = 12;

// Hash password
const hash = await bcrypt.hash(password, saltRounds);

// Verify password
const match = await bcrypt.compare(password, hash);

// Input Validation with Joi
const Joi = require('joi');

const schema = Joi.object({
  email: Joi.string().email().required(),
  password: Joi.string().min(8).max(30).required(),
  username: Joi.string().alphanum().min(3).max(30).required()
});

const { error, value } = schema.validate(req.body);

// Express Security Headers
const helmet = require('helmet');
app.use(helmet());

// Rate Limiting
const rateLimit = require('express-rate-limit');
const limiter = rateLimit({
  windowMs: 15 * 60 * 1000, // 15 minutes
  max: 100 // limit each IP to 100 requests per windowMs
});
app.use(limiter);

9. Testing Best Practices

  • Write unit tests - Test individual functions/components
  • Use integration tests - Test component interactions
  • Implement E2E tests - Test complete user flows
  • Use test coverage tools - Aim for 80%+ coverage
  • Mock external dependencies - Isolate units for testing
  • Write descriptive test names - Document expected behavior
  • Use TDD when appropriate - Write tests before code
// Jest Unit Test Example
describe('UserService', () => {
  describe('createUser', () => {
    it('should create a new user with valid data', async () => {
      const userData = {
        email: 'test@example.com',
        username: 'testuser'
      };
      
      const user = await UserService.createUser(userData);
      
      expect(user).toBeDefined();
      expect(user.email).toBe(userData.email);
      expect(user.id).toBeDefined();
    });
    
    it('should throw error for duplicate email', async () => {
      await expect(
        UserService.createUser({ email: 'existing@example.com' })
      ).rejects.toThrow('Email already exists');
    });
  });
});
